Adventure Game (1200 LOC)

March - April 2021
Applies prime concepts of OOP. Combat-based, player progresses through the game, fights different types of chracters, has an inventory, can upgrade abilites and stats, all through a detailed GUI. Fighting is turn based and the attack values are determined an algorithm which uses an RNG and that character's stats. Each character has different dialogue, fighting-style and stats. Futhermore, every character has a special attack. These features were outline in the abstract super class 'character' and overriden in all subclasses. Lead to a final grade of >90%.

'Character' is a abstract superclass for all other character-types. The enemies that the user encounters and in which in order is partially randomized. As the character progresses they improve abilites and statistics. They also pick up items which are stored in their inventory, implemented with an arraylist. Items in the inventory have their own hierarchy with an interface also. The player can save the state of the game and load from a previous save, all through a detailed GUI. The GUI consists of buttons to interact (e.g. Attack, Heal etc) as well as a dynamic health bar which changes coolour depending on how much health the player and enemy have.

The Liskov Subsitiution principle was very useful in this project is it allowed generalisation and reuse of code in multiple places, as did generic methods.

  • Full Exception Handling
  • Interfaces
  • ArrayLists
  • Generic Types
  • Inheritance, Overriding, Polymorphism
  • Allows player to Save and Load Progress (File Handling)
  • Detailed GUI
  • Written in Java
